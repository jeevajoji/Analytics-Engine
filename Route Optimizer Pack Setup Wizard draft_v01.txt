Create New Operation Wizard:

Step 1: Type Selection
- Small Grids with categories

Step 2: Basic Details
Page Title: Operation Details
Form:
Operation Name - Text - Required, user-friendly identifier
Description - Text - Operation description
Timezone - Auto-detected but editable - Helpful for session windows & timestamps

Step 3: Units & Sessions
Page Title: Fleet Selection & Pickup Sessions
A. Select Units (Buses)
Multi-select with a searchable list
Show attributes: Bus Number, Capacity, Assigned Driver
B. Add Sessions
User creates any number of sessions (Using add new row approach)
Each session has:
- Session Name
- Start Window
- End Window

Step 4: Monitoring
Page Title: Monitoring Levels
A. Expandable sections (multiselectable): 
- Trip Cycle
- Session Cycle
- Daily Cycle
- Weekly Cycle
- Monthly Cycle
- Quarterly Cycle
- Yearly Cycle
Options inside each sections:
- Provisional Window: Number of trips / Number of days
- Baseline Window: Number of trips / Number of days
- Refresh Frequency
B. Information Tooltip:
Longer windows = higher confidence. Short windows = faster adaptation.

Step 5: Optimization Mode
Page Title: Objective Selection
A. Primary Objective (Mandatory, Single select):
- Time
- Distance
- Balancing
B. Secondary Objectives (Optional, Multi-select):
- Reduce Number of Stops
- Minimize Overlaps Between Buses
- Maximize Student Safety Score (crossings, U-turns, etc.)
- Maximize Route Consistency (same pickup order daily)

Step 6: Rules & Constraints (Post MVP)
Page Title: Logistics & Compliance Rules
- Max ride duration/distance
- Capacity enforcement / buffers
- Pickup window (Destination mapping or Map-based - Points or Polygon selection)
- Restricted Roads / No-Go Zones (Map-based, polygon selection)
- Campus entry/exit sequence
